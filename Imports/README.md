# Imports

Crimes against namespaces, dependency graphs, and knowing what's currently bound to the name `print` in your own file.

- `import_antigravity.py` — a real stdlib easter egg
- `_wildcard_source.py` + `star_import_chaos.py` — a module engineered to poison anything that star-imports it
- `circular_a.py` + `circular_b.py` + `run_circular.py` — two modules that import each other, and survive by deferring the import
- `import_inside_loop.py` — importing the same module 100,000 times, on purpose
